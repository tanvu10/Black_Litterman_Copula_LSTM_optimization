# Black-Litterman Copula LSTM Optimization

Research-oriented portfolio construction pipeline combining:
- deep learning return views (LSTM)
- copula-based covariance estimation (R)
- Black-Litterman posterior blending
- constrained max-Sharpe optimization

The project targets VN30-style equity universes and compares baseline vs copula-enhanced Black-Litterman strategies.

## Why this project matters

This repository demonstrates an end-to-end quantitative research workflow:
1. feature engineering and return data preparation
2. rolling ML forecast generation
3. copula risk modeling
4. portfolio optimization and backtesting
5. parameter sensitivity analysis

It is structured to support reproducibility, ablation, and resume-ready reporting.

## Repository structure

- `main.py`: unified CLI entry point for the full pipeline.
- `utility.py`: reusable quantitative primitives (indicators, BL posterior, optimizers, metrics).
- `data_processing.py`: prepares model-ready features and aligned return matrices.
- `LTSM_train.py`: LSTM forecaster for one-step return prediction.
- `DL_train_processing.py`: rolling prediction generation across assets.
- `BL_optimization.py`: configurable backtest runner and performance output.
- `analysis_1.py`: descriptive statistics for asset returns.
- `analysis_2.py`: BL sensitivity analysis across `tau`, `delta`, and confidence levels.
- `CBL_R_processing/`: R notebooks for residual extraction and copula covariance generation.

## Methodology

### 1. Return view modeling
For each stock, technical and factor-linked features are used to train an LSTM model that predicts next-day return.

### 2. Copula covariance modeling
Residuals are modeled with copulas (Gaussian, Clayton, Frank, Gumbel) to produce alternative covariance matrices.

### 3. Black-Litterman posterior
Predicted returns are treated as views and combined with market-implied equilibrium returns:

- Prior: copula-derived covariance (or empirical covariance)
- Views: LSTM return forecasts
- Posterior outputs: `mu_BL`, `cov_BL`

### 4. Portfolio optimization
Long-only, constrained max-Sharpe optimization (`SLSQP`) with weight caps (`upper_bound`) is applied for each strategy.

### 5. Evaluation
Outputs include:
- strategy return series
- annualized Sharpe ratio
- average drawdown
- max drawdown
- parameter sensitivity tables

## Setup

1. Create a Python 3.9+ environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Put input datasets under `data_v2/` (or pass a custom data root).

Expected key files under `data_v2/`:
- `factor_return_df.csv`
- `mkt_cap_weight.csv`
- `stock_return_df.csv`
- `stock_prediction.csv` (generated by pipeline)
- `stock_data_folder/*.csv`
- `sub_factor_folder/*.csv`
- `cov_matrix/{Gauss,Clayton,Frank,Gumbel}/*.csv`

## Usage

Run commands through `main.py`.

### Prepare engineered datasets
```bash
python main.py prepare-data --data-root data_v2
```

### Generate rolling LSTM predictions
```bash
python main.py predict --data-root data_v2 --start-date 2021-10-01 --end-date 2022-04-01
```

### Run backtest
```bash
python main.py backtest --data-root data_v2 --output-dir outputs --upper-bound 0.5 --tau 0.01 --delta 2.5 --confidence-level 0.9 --plot
```

### Descriptive statistics
```bash
python main.py describe --data-root data_v2 --output outputs/descriptive_stats.csv
```

### Sensitivity analysis
```bash
python main.py sensitivity --data-root data_v2 --output-root outputs/sensitive_check --upper-bound 0.5
```
